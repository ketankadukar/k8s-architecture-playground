This diagram shows how the control plane and worker nodes interact to manage workloads.

API Server

The API Server is the gateway to the cluster. It validates and processes requests from kubectl, controllers, and external clients.

etcd

etcd is a distributed key-value store that holds the entire cluster state, including pods, deployments, and configurations.

Admission Controllers

Admission Controllers enforce policies and can mutate or validate objects before they are persisted in etcd.

Scheduler

The Scheduler assigns unscheduled pods to nodes based on resource availability, constraints, and policies.

Controller Manager

The Controller Manager runs controllers that reconcile desired state with actual state, ensuring workloads are maintained.

Worker Nodes

Worker nodes run application workloads. Each node has a kubelet (agent), container runtime, and kube-proxy for networking.
